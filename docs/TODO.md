# TODO — Improvement Backlog (Wave 7+)

Prioritized backlog from the 2026-07-03 full-codebase review. Items are grouped into six buckets ordered by leverage on user outcomes. Bucket 1 (statistical correctness) takes precedence — a polished agent that misreports a significance call is worse than a rougher agent that calls it correctly. Bucket 2 (backend unification) is the structural fix that retires most of Bucket 1's divergence bugs permanently.

The previous backlog (items 1–35, waves 1–6, all shipped) is archived at [TODO-waves-1-6.md](TODO-waves-1-6.md). Numbering here continues at 36 so `git log --grep "TODO.md #N"` remains unambiguous.

Effort tags: **(S)** ≤ ½ day, **(M)** 1–3 days, **(L)** > 3 days.

## Bucket 1 — Statistical correctness

Highest leverage. These items affect what the agent tells users about their experiments.

36. [ ] **Fix effect-size / confidence-interval scale mismatch for GLM families** — for binary/count/heavy-tail metrics, `effect_size` is the raw mean/proportion difference (`model_families.py:230`) but `confidence_interval` is the GLM term CI in log-odds/log-rate units (`model_families.py:473`). Charts draw log-odds error bars around a raw difference (`charts_core.py:59-85`). Report the CI on the same scale as the effect (marginal effect / rate difference), or carry both scales explicitly on `ABTestResult`. _Files:_ `src/statistics/model_families.py`, `src/statistics/models.py`, `src/statistics/charts_core.py`. **(M)**
37. [ ] **Align proportion-test method across backends** — pandas uses `method="score"` (`statsmodels_engine.py:480`), Spark uses `method="wald"` (`pyspark_analyzer.py:695`); same data yields different p-values. Standardize on score and port the pandas guardrails (small-n, expected-cell-count, degenerate inputs) into the Spark path. _Files:_ `src/statistics/pyspark_analyzer.py`, `src/statistics/statsmodels_engine.py`, `tests/test_parity_pandas_spark.py`. **(M)**
38. [ ] **Spark: honor `sequential_config` and multiple-testing correction, or fail loudly** — Spark accepts `sequential_config` and silently ignores it (`pyspark_analyzer.py:979`, `:1186`) and skips BH/FDR in segmented analysis. Apply both, and raise a structured error for any remaining unsupported option instead of returning defaulted fields. _Files:_ `src/statistics/pyspark_analyzer.py`. **(M)**
39. [ ] **Replace observed-power adequacy with MDE-based adequacy** — `is_sample_adequate` is computed from post-hoc power on the realized effect (`analyzer.py:466-473`, `:720`; `pyspark_analyzer.py:1046-1048`), the classic observed-power fallacy. Key adequacy off `achieved_mde` (already computed at `analyzer.py:766`) vs. a target effect instead. _Files:_ `src/statistics/analyzer.py`, `src/statistics/pyspark_analyzer.py`, `src/agent_reporting.py`. **(S)**
40. [ ] **Allocation-ratio-aware SRM** — the SRM check hardcodes an expected 50/50 split (`diagnostics.py:94`, `statsmodels_engine.py:266`, `pyspark_analyzer.py:836`), so an intentional 90/10 holdout trips SRM and silently forces `is_significant=False` (`analyzer.py:524-529`). Add `expected_treatment_ratio` to config / `run_ab_test` and thread it through. _Files:_ `src/statistics/diagnostics.py`, `src/statistics/statsmodels_engine.py`, `src/statistics/pyspark_analyzer.py`, `src/config.py`. **(M)**
41. [x] **Close the deterministic-difference significance leak** — the zero-variance branch returns `p_value=0.0` with `blocks_significance=False` regardless of `small_n` (`model_families.py:278-311`), so two constant-but-unequal points per arm read as significant. Honor `small_n` like every other branch. _Files:_ `src/statistics/model_families.py`. **(S)**
42. [ ] **Fix `total_effect` double-counting** — the combined effect adds the t-test total effect to the proportion effect (`analyzer.py:566-576`; `pyspark_analyzer.py:1077-1085`), overstating impact whenever both tests are significant (the mean difference already includes value from incremental converters). Fix the arithmetic or document the narrow metric construction where addition is valid. _Files:_ `src/statistics/analyzer.py`, `src/statistics/pyspark_analyzer.py`, `src/statistics/summary_builder.py`. **(M)**
43. [ ] **Surface silent statistical fallbacks** — model-fit exceptions silently fall back OLS→CompareMeans (`model_families.py:419-445`), proportion-test exceptions resolve to `p=1.0` (`statsmodels_engine.py:509`; `pyspark_analyzer.py:701`), and power/MDE/required-N exceptions return `0.0`/sentinels (`power_analysis.py:56,87,114`). A systematically failing model currently looks like a clean null. Propagate structured warnings into the rendered report. _Files:_ `src/statistics/model_families.py`, `src/statistics/power_analysis.py`, `src/agent_reporting.py`. **(M)**
44. [ ] **Numeric-precision cleanups** — use `scipy` survival functions instead of `1 - cdf` for tail p-values (`pyspark_analyzer.py:660`, `:711`); estimate NegativeBinomial dispersion instead of hardcoding `alpha=1.0` (`model_families.py:373`); unify the required-N sentinel (`int(1e9)` at `power_analysis.py:100` vs `999999` at `pyspark_analyzer.py:739`). _Files:_ `src/statistics/pyspark_analyzer.py`, `src/statistics/model_families.py`, `src/statistics/power_analysis.py`. **(S)**

## Bucket 2 — Backend unification

The structural fix. Most Bucket 1 divergences exist because the two backends re-implement the same statistics independently.

45. [ ] **Extract a shared statistical core** — Spark hand-rolls Cohen's d, power, required-N, Bayesian Monte Carlo, and the proportion test (`pyspark_analyzer.py:621-800`), duplicating `power_analysis.py` / `bayesian.py` / `statsmodels_engine.py`. The Spark path already collects small aggregates to the driver (`pyspark_analyzer.py:1003`), so both backends can call the same backend-agnostic helpers taking `(mean, var, n)` or arrays. Deletes roughly `pyspark_analyzer.py:620-1180` and makes answer divergence structurally impossible. Land after #37/#38 so parity tests pin behavior first. _Files:_ `src/statistics/pyspark_analyzer.py`, `src/statistics/analyzer.py`, `src/statistics/power_analysis.py`, `src/statistics/bayesian.py`, `tests/test_parity_pandas_spark.py`. **(L)**
46. [ ] **Restructure `ABTestResult` and consolidate serialization** — the ~90-field flat dataclass (`models.py:70-182`) forces enormous keyword assembly at three call sites, and the hand-written 80-key `to_legacy_dict` (`models.py:184-273`) plus `canonical_result_as_dict` plus `SparkABTestResult.to_serializable_dict` (`pyspark_analyzer.py:127`) define the alias vocabulary three times. Group fields into nested sub-models (frequentist / proportion / bayesian / sequential / diagnostics) and generate legacy aliases from `_CANONICAL_FIELD_ALIASES`. _Files:_ `src/statistics/models.py`, `src/statistics/analyzer.py`, `src/statistics/pyspark_analyzer.py`. **(L)**
47. [ ] **Centralize numeric configuration** — Bayesian decision thresholds are hardcoded at 0.95/0.05 and ignore `significance_level` (`analyzer.py:588-589`; `charts_extended.py:36-38`); charts hardcode `p<0.05` and 80% power labels (`charts_core.py:214,242,292`; `charts_dashboard.py:243`); magic thresholds for heavy-tail detection, overdispersion, variance ratio, outlier sensitivity, and the n>30 normal cutoff are scattered literals (`model_families.py:77-86,367`; `diagnostics.py:315`; `bayesian.py:81`). Name them as constants on `Config`/engine and thread alpha/power everywhere they render. _Files:_ `src/config.py`, `src/statistics/*`. **(M)**
